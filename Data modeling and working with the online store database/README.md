# Logical Data Model and Data Dictionary

A project focused on updating the requirements specification for a web application.

## Role

System Analyst.

## Completed Tasks

* analyzed the existing requirements specification;
* created a logical data model;
* designed an ER diagram using Crow’s Foot notation;
* normalized the data model to Third Normal Form;
* described entities, relationships, and key attributes;
* prepared a data dictionary;
* added the data model and data dictionary to the updated requirements specification.

## Artifacts

* requirements specification;
* logical data model;
* [ER diagram](https://github.com/Alionkor/System-analyst-Yandex-Practicum/blob/main/Data%20modeling%20and%20working%20with%20the%20online%20store%20database/photo_2026-07-05_13-29-01.jpg);
* [data dictionary](https://github.com/Alionkor/System-analyst-Yandex-Practicum/blob/main/Data%20modeling%20and%20working%20with%20the%20online%20store%20database/data-dictionary.png).

## Tools and Notations

* ER modeling;
* Crow’s Foot;
* Third Normal Form;
* Google Docs;
* diagrams.net.

## Result

An updated version of the requirements specification was prepared and supplemented with a logical data model and a data dictionary. The materials can be used by the development team to design the database structure and clarify the application’s data requirements.
