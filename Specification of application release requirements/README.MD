# Спецификация требований для второго релиза приложения

## Описание

Проект посвящён доработке спецификации требований для второго релиза приложения в сфере изучения иностранных языков.

В рамках работы были описаны новые функциональные возможности, обновлены пользовательские сценарии и подготовлены требования для команды разработки.

## Моя роль

Системный аналитик.

## Что было сделано

* описаны функциональные требования для функции карточек;
* обновлена UML Use Case диаграмма;
* добавлены табличные варианты использования для новой функции;
* описаны функциональные требования для просмотра видео;
* сформулированы атрибуты качества;
* описаны бизнес-правила;
* дополнен глоссарий;
* выполнена проверка требований на полноту и согласованность.

## Артефакты

* спецификация требований;
* UML Use Case диаграмма;
* табличные варианты использования;
* функциональные требования;
* атрибуты качества;
* бизнес-правила;
* глоссарий.

## Инструменты и нотации

* Google Docs;

## Результат

Подготовлена обновлённая спецификация требований для второго релиза приложения. Документ содержит описание новых функций, сценариев использования, функциональных требований, бизнес-правил, атрибутов качества и глоссария.

# Requirements Specification for the Second Application Release

## Description

This project focuses on updating the software requirements specification for the second release of an application in the foreign language learning domain.

As part of the project, new functional capabilities were described, user scenarios were updated, and requirements were prepared for the development team.

## My Role

System Analyst.

## What Was Done

* described functional requirements for the flashcards feature;
* updated the UML Use Case diagram;
* added tabular use case descriptions for the new feature;
* described functional requirements for the video viewing feature;
* defined quality attributes;
* described business rules;
* updated the glossary;
* checked the requirements for completeness and consistency.

## Artifacts

* software requirements specification;
* UML Use Case diagram;
* tabular use case descriptions;
* functional requirements;
* quality attributes;
* business rules;
* glossary.

## Tools and Notations

* UML Use Case;
* functional requirements;
* Use Case Specification;
* Google Docs;
* diagrams.net.

## Result

An updated requirements specification for the second application release was prepared. The document includes descriptions of new features, use cases, functional requirements, business rules, quality attributes, and a glossary.
